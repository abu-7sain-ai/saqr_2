# مطبخ الخبراء (The Expert Kitchen)

## الفكرة الأساسية

المطبخ هو "مركز أبحاث" يُدار بالذكاء الاصطناعي. المستخدم يضغط زر واحد، وتبدأ عملية تحليل شاملة تنتهي بتوليد موظفين آليين جاهزين للتداول التجريبي.

---

## المحفّز (Trigger)

يمكن بدء جلسة المطبخ بطريقتين:

1. **من الموقع**: زر "تشغيل المطبخ" في صفحة اجتماع الخبراء.
2. **من التلغرام**: إرسال أمر "ابدأ اجتماع" للبوت.

### مدخلات الجلسة

| المدخل | الوصف | القيم |
| :--- | :--- | :--- |
| نوع السوق | حالة السوق الحالية | مستقر / متوتر |
| رأس المال | المبلغ المخصص لموظفي هذه الجلسة | رقم بالدولار |
| المنصة | منصة التداول المستهدفة | Binance / Coinbase / إلخ |

---

## جمع البيانات (Data Collection)

بمجرد بدء الجلسة، يقوم Python بجمع البيانات التالية أوتوماتيكياً:

```
OHLC آخر 3-6 أشهر (4H و 15M)
Fear & Greed Index الحالي
أخبار CryptoPanic
Bitcoin Dominance
Twitter (عبر Grok)
أداء الموظفين الحاليين
سجل الاستراتيجيات الفاشلة
قرارات المجلس السابقة
```

## جولات النقاش (The 4 Rounds)

### الجولة الأولى — تحليل مستقل

كل خبير يحلل البيانات بشكل مستقل دون معرفة آراء الآخرين:

| الخبير | الموديل | المهمة | المدخلات | المخرجات |
| :--- | :--- | :--- | :--- | :--- |
| **الشارتيست** | Qwen | التحليل الفني | شارت 4H + 15M + RSI + MACD + ATR + Bollinger | نقطة دخول + هدف + وقف خسارة + قوة الإشارة |
| **المذيع** | Grok | تحليل الأخبار | أخبار Twitter + CryptoPanic | توصية (ادخل/انتظر/اخرج) + أهم خبر + تأثيره |
| **النبّاض** | Grok | تحليل المشاعر | Fear & Greed + Twitter + حجم التداول | هل الوقت مناسب نفسياً؟ |
| **الرادار** | Qwen | تحليل السوق الكلي | Bitcoin Dominance + حجم السوق | موسم Bitcoin أم Altcoins؟ + قوة السوق |
| **الحارس** | DeepSeek | تحليل المخاطر | كل البيانات + سجل الفشل | أكبر 3 مخاطر + مقبول/مرفوض |
| **المحقق** | Claude | توحيد القواعد | آراء الخبراء الخمسة | قواعد برمجية واضحة (if/then) |

### الجولة الثانية — الردود المتقاطعة

كل خبير يطّلع على آراء زملائه ويُعدّل أو يؤكد رأيه:

```
الشارتيست يرد على المذيع والنبّاض
المذيع يرد على الشارتيست والرادار
النبّاض يرد على الشارتيست والحارس
الرادار يرد على الكل
الحارس يرد على الكل بتشكيك
المحقق يحدّث قواعده البرمجية
```

### الجولة الثالثة — اعتراض الحارس

```
Python يرسل القرار الأولي للحارس:
"هذه الاستراتيجية المقترحة — ابحث عن كل أسباب فشلها"

الحارس يعترض بقوة
الخبراء يردون على اعتراضاته
المحقق يوثّق نقاط الخلاف
```

### الجولة الرابعة — الأمير يحسم

```
الأمير (DeepSeek) يقرأ كل النقاش كاملاً

أولوياته:
  1. الحارس في المخاطر
  2. الشارتيست في التوقيت
  3. المذيع في الأخبار

يصدر 3 استراتيجيات فقط:
```

**الاستراتيجية 1 — محافظة:**
- شرط الدخول: RSI < 35 و MACD يتقاطع و Fear < 40
- الهدف: +2%
- وقف الخسارة: -1%
- حجم الصفقة: 0.5% من المحفظة

**الاستراتيجية 2 — متوسطة:**
- شرط الدخول: RSI < 40 و MACD يتقاطع
- الهدف: +3%
- وقف الخسارة: -1.5%
- حجم الصفقة: 1% من المحفظة

**الاستراتيجية 3 — جريئة:**
- شرط الدخول: RSI < 45 و Bollinger يضيق
- الهدف: +5%
- وقف الخسارة: -2%
- حجم الصفقة: 1.5% من المحفظة

---

## المهندس يكتب الكود

```
يستلم 3 استراتيجيات من الأمير
        ↓
يكتب Python لكل استراتيجية:
  - منطق الدخول
  - حساب التكاليف (انزلاق + عمولة + spread)
  - Trailing Stop Loss
  - تسجيل Supabase
  - إشعار تليجرام
        ↓
يختبر الكود → يصلح الأخطاء
        ↓
لو فشل الإصلاح → يرسل تقرير للأمير
```

---

## Backtesting

```
Python يختبر كل استراتيجية على 10 سنوات (yfinance)
مع التكاليف الحقيقية (انزلاق + عمولة + spread)

معايير النجاح:
  Sharpe > 1.0 ✓
  Drawdown < 15% ✓
  فوز > 50% ✓

فشلت؟ → ترجع للأمير مع تقرير السبب
نجحت؟ → Walk-Forward Testing
```

---

## Walk-Forward Testing

```
سنة 1-7 → Python يبني الاستراتيجية
سنة 8-10 → Python يختبر على بيانات جديدة

فشلت؟ → ترجع للأمير
نجحت؟ → صنع الموظفين
```

---

## ولادة الموظفين

كل استراتيجية تولّد نسختين (محافظة + جريئة) = **6 موظفين**:

```
الاستراتيجية 1 المحافظة → موظف 1
الاستراتيجية 1 الجريئة  → موظف 2
الاستراتيجية 2 المحافظة → موظف 3
الاستراتيجية 2 الجريئة  → موظف 4
الاستراتيجية 3 المحافظة → موظف 5
الاستراتيجية 3 الجريئة  → موظف 6
```

كل موظف يُنشأ في Supabase بملف كامل:
- الاسم والرقم التسلسلي
- النوع: `paper`
- الاستراتيجية (من المطبخ)
- الإعدادات (رأس المال + حجم الصفقة + عدد الصفقات)
- `kitchen_session_id` ← مرتبط بجلسة المطبخ

**يبدأون العمل تلقائياً في بيئة Paper Trading.**

## الإشعار

عند الانتهاء، يرسل البوت إشعار تلغرام:

```
"تم إنشاء 6 موظفين جدد من جلسة المطبخ رقم X
موظف 1 إلى موظف 6 جاهزون للتشغيل"
```

## تخزين البيانات

- كل جولات النقاش تُخزن في حقل واحد `expert_opinions` (JSONB) بنظام حوار متسلسل: "فلان قال كذا → فلان رد كذا".
- الموديلات تُستدعى عبر **OpenRouter** بمفتاح واحد.
- النظام يعمل في الخلفية بالكامل ولا يتطلب بقاء المستخدم متصلاً.
