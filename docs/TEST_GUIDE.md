# دليل استخدام نظام الجودة (Saqr Test Guide) 🛡️

## 0. جهاز التحكم السريع (One-Click Control) 🪄
إذا كنت لا تريد كتابة أي أوامر، اذهب للمجلد الرئيسي وافتح ملف:
**`SAQR_SHIELD.bat`** (اضغط عليه مرتين بالماوس).
سيقوم هذا الملف بتشغيل كل الاختبارات وتنظيف الكود آلياً وإعطائك تقريراً نهائياً.

---

## 1. فحص المنطق الداخلي (Unit Testing - Vitest)
يستخدم هذا الفحص للتأكد من أن الحسابات والبرمجيات خلف الكواليس تعمل بدقة.
*   **كيف تشغله؟**: اكتب مباشرة في التيرمينال:
    ```bash
    npm run test:unit
    ```
*   **ماذا تتوقع؟**: ستظهر لك قائمة بالاختبارات التي تمت، وستراها باللون الأخضر ✅ إذا كان كل شيء سليماً.

## 2. فحص الخلفية والمخ البرمجي (Backend Testing - Pytest)
يستخدم هذا الفحص للتأكد من أن "مخ" الصقر (كود البايثون) سليم ولا يحتوي على أخطاء برمجية أو نقص في المكتبات.
*   **كيف تشغله؟**: اكتب مباشرة في التيرمينال:
    ```bash
    npm run test:backend
    ```
*   **ماذا تتوقع؟**: سيقوم النظام بفحص ملفات `factory.py` و `analytics_service.py` وغيرها، وسيعطيك `passed` إذا كانت سليمة.

## 3. فحص رحلة المستخدم (UI Testing - Playwright)
هذا الفحص يحاكي مستخدماً حقيقياً يفتح الموقع ويضغط الأزرار.
*   **كيف تشغله؟**: اكتب مباشرة في التيرمينال:
    ```bash
    npm run test:ui
    ```
*   **ماذا تتوقع؟**: سيفتح النظام متصفحاً آلياً (مخفياً)، يقوم بتسجيل الدخول، يذهب لصفحات الموقع، ويتأكد أن كل شيء يظهر بشكل صحيح.

## 4. تنظيف وترتيب الكود (Prettier)
للحفاظ على كود نظيف واحترافي بنفس التنسيق في كل الملفات.
*   **كيف تشغله؟**: اكتب مباشرة في التيرمينال:
    ```bash
    npm run format
    ```

## 5. الحارس (Husky)
هذا النظام يعمل **تلقائياً**. إذا حاولت حفظ أي كود يحتوي على أخطاء برمجية، فسيقوم "الحارس" بمنع العملية ويطلب منك إصلاح الخطأ أولاً لضمان عدم تلف المشروع.

---
**تذكر**: هذه الاختبارات هي "صمام الأمان" لمشروعك. يفضل تشغيلها دائماً قبل نشر أي تحديث جديد للنسخة الحية.
