# دستور منصة صقر (System Specification) - v3.0

هذا الملف هو المرجع الأعلى والوحيد لكل ميزات المنصة، صُمم لضمان أن كل سطر كود يخدم أهداف "صقر" الاستراتيجية.

## 1. الهوية والهدف (The Vision)
منصة "صقر" هي نظام تداول آلي كوانتي (Quantitative) يعتمد على مجلس حكماء من الذكاء الاصطناعي لاتخاذ قرارات تداول مبنية على بيانات تاريخية (10 سنوات).

## 2. المكونات الرئيسية (Core Components)

### أ. اجتماع الخبراء (The Kitchen)
*   **الوظيفة**: محرك توليد الاستراتيجيات.
*   **الجولات**: 7 جولات من النقاش العلمي (تشريح البيانات -> فرضيات -> هجوم الفريق الأحمر -> تنقيح -> ببعة سوداء -> تدقيق -> مرسوم الأمير).
*   **المخرجات**: استراتيجيات (Conservative, Moderate, Aggressive) اجتازت اختبار الـ 7+3 سنوات.

### ب. مركز العمليات (Operations Center)
*   **المستشار (Advisor)**: نظام رقابي صامت، يقدم تحليلاً تنفيذياً عند الطلب فقط (On-Demand).
*   **لوحة التحكم**: عرض حي لسلامة المنظومة، الرصيد المتاح في OpenRouter، وحالة الربط مع المنصات.

### ج. الموظفون (The Workers)
*   **الأنواع**: موظف عادي (Standard)، موظف مطور (Advanced/Learning)، موظف قناص (Hunter).
*   **التشغيل**: يتم استنساخ الموظفين مباشرة من نتائج اجتماع الخبراء لضمان دقة التنفيذ.

### د. الأسواق المتاحة (Market Management)
*   **القائمة البيضاء (Whitelist)**: قائمة العملات المسموح بتداولها.
*   **الربط (APIs)**: إدارة مفاتيح API الخاصة بـ Binance وخدمات جلب البيانات.

### هـ. التحليلات والأداء (Analytics 2.0)
*   **المصفوفة الشهرية**: مقارنة أداء الموظفين شهرياً بالدولار والنسبة المئوية.
*   **داشبورد التوكنات**: مراقبة تكلفة استهلاك عقول الذكاء الاصطناعي (Claude, Grok, DeepSeek).
*   **صحة النظام (Grafana style)**: مراقبة حية لـ CPU و RAM مع تنبيهات تليجرام تلقائية عند الخطر.

## 3. قواعد الأمان والرقابة
*   **القائمة البيضاء الصارمة**: يمنع التداول في أي عملة خارج القائمة.
*   **تنبيهات تليجرام**: تقارير فورية عن (دخول صفقات، خروج، فشل باكتست، استهلاك موارد السيرفر).
*   **تصفير السيولة**: نظام آلي لتقليص الصفقات عند انخفاض الرصيد الحر.
