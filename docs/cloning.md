# نظام الاستنساخ (Worker Cloning System)

## الفكرة الأساسية

الاستنساخ = نسخ موظف بنفس الاستراتيجية (فلسفة الدخول والخروج) لكن بإعدادات مالية مختلفة.

**الاستراتيجية هي "الحمض النووي" الثابت. الإعدادات المالية هي "البيئة" المتغيرة.**

---

## أنواع الاستنساخ

### 1. Paper → Paper (تجربة إعدادات جديدة)

**الغرض**: تجربة نفس الاستراتيجية بظروف مالية مختلفة دون مخاطرة.

```
المصدر: جندي مشاة الأمير 1 (Paper)
  استراتيجية: RSI < 35 + MACD يتقاطع
  رأس المال: $500
  حجم الصفقة: 1%
  نجح 3 أشهر ✅
        ↓
الاستنساخ: جندي مشاة الأمير 1-B (Paper)
  استراتيجية: RSI < 35 + MACD يتقاطع  ← نفسها بالضبط
  رأس المال: $2000                      ← تغيّر
  حجم الصفقة: 2%                        ← تغيّر
```

**ما يتغير**: رأس المال، نسبة المخاطرة، عدد الصفقات، حجم الصفقة.
**ما يبقى ثابتاً**: شروط الدخول، شروط الخروج، وقف الخسارة، الهدف.

---

### 2. Paper → Live (ترقية للتداول الحقيقي)

**الغرض**: نقل موظف أثبت كفاءته من البيئة الوهمية إلى التداول بأموال حقيقية.

```
المصدر: جندي مشاة الأمير 1 (Paper)
  نجح في:
    ✓ Backtesting (10 سنوات)
    ✓ Walk-Forward Testing
    ✓ Paper Trading (3+ أشهر)
        ↓
الاستنساخ: انتحاري مشاة الأمير 1 (Live)
  نفس الاستراتيجية بالضبط
  نفس الإعدادات (أو إعدادات مختلفة)
  لكن بفلوس حقيقية 💰
```

**شرط الترقية**: المستخدم يراجع الأداء يدوياً ويضغط "استنساخ للحقيقي".

---

### 3. Live → Live (تنويع المحفظة الحقيقية)

**الغرض**: تشغيل نفس الاستراتيجية الناجحة بإعدادات مالية مختلفة لتنويع المخاطر.

```
المصدر: انتحاري مشاة الأمير 1 (Live)
  استراتيجية: RSI < 35 + MACD يتقاطع
  رأس المال: $1000
  يربح منذ شهر ✅
        ↓
الاستنساخ: انتحاري مشاة الأمير 1-B (Live)
  نفس الاستراتيجية بالضبط
  رأس المال: $3000  ← تغيّر
```

---

## مصفوفة الاستنساخ (Cloning Matrix)

| المصدر | الهدف | ما يتغير | ما يبقى | الغرض |
| :--- | :--- | :--- | :--- | :--- |
| Paper | Paper | الإعدادات المالية | الاستراتيجية | تجربة ظروف مختلفة |
| Paper | Live | النوع (+ إعدادات اختيارياً) | الاستراتيجية | ترقية موظف مُثبت |
| Live | Live | الإعدادات المالية | الاستراتيجية | تنويع المحفظة |

---

## الإعدادات القابلة للتغيير عند الاستنساخ

| الإعداد | الوصف | مثال |
| :--- | :--- | :--- |
| رأس المال | المبلغ المخصص للموظف | $500 → $2000 |
| حجم الصفقة | نسبة المبلغ لكل صفقة | 1% → 2% |
| عدد الصفقات المتزامنة | كم صفقة يفتح بنفس الوقت | 1 → 3 |
| نسبة المخاطرة | الحد الأقصى للخسارة اليومية | 2% → 5% |

---

## الإعدادات الثابتة (الحمض النووي)

| الإعداد | الوصف |
| :--- | :--- |
| شروط الدخول | متى يدخل الصفقة (RSI, MACD, إلخ) |
| شروط الخروج | متى يخرج من الصفقة |
| وقف الخسارة | نسبة الخسارة المقبولة لكل صفقة |
| الهدف | نسبة الربح المستهدفة لكل صفقة |
| `kitchen_session_id` | مرجع جلسة المطبخ التي أنتجت هذه الاستراتيجية |

## التنفيذ التقني

### في قاعدة البيانات (Supabase)

عند الاستنساخ، يتم إنشاء سجل جديد في جدول `workers` مع:

```
- id: UUID جديد
- name: اسم جديد (مع لاحقة -B أو -C)
- strategy: نسخة من strategy الأصلي (بدون تعديل)
- settings: الإعدادات الجديدة التي حددها المستخدم
- type: paper أو live (حسب نوع الاستنساخ)
- cloned_from: UUID الموظف الأصلي (للتتبع)
- kitchen_session_id: نفس جلسة المطبخ الأصلية
- status: running (يبدأ العمل فوراً)
```

### في الواجهة (React)

- زر **"استنساخ"** في بطاقة كل موظف.
- نافذة منبثقة (Modal) للاختيار:
  - نوع الهدف: Paper أو Live
  - تعديل الإعدادات المالية
- تأكيد بسيط قبل الاستنساخ للـ Live (حماية من الضغط بالخطأ).
